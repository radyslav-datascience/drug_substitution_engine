# `_optional_calculations/` — namespace для ad-hoc розрахунків

Ця папка призначена для **специфічних запитів замовника**, які НЕ є частиною
основної виробничої методології обчислення коефіцієнтів субституції.

## Архітектурне правило

**Продакшн-пайплайн** (`pipeline/`, `core/`, `config/`) реалізує математично
обґрунтовані розрахунки і **не змінюється** під ad-hoc запит замовника.
Пайплайн може розширюватися новими методологічними фічами (нові метрики,
покращення алгоритмів) — але такі зміни підпорядковуються data-science
обґрунтуванню, а не бізнес-побажанням разового характеру.

**Ad-hoc розрахунки** (фільтри, top-N виборки, кастомні зрізи, comparison-
аналіз) живуть **тут**, у виокремленій папці `_optional_calculations/`.

## Структура

Кожна задача отримує **окрему підпапку** з говорящою назвою:

```
_optional_calculations/
├── README.md                              ← цей файл
└── <task_name>/
    ├── README.md                          ← опис задачі та логіки
    ├── inputs/                            ← вхідні файли (sales, partners, тощо)
    ├── outputs/                           ← результати
    ├── logs/                              ← логи запусків
    └── run_<task>.py                      ← скрипт обробки
```

## Поточні задачі

| Папка | Опис | Статус |
|-------|------|--------|
| `top_1k_reliability_sales_volume/` | Top-N препаратів за обсягом продажів × надійністю COEF_1 (HIGH/MEDIUM only) | 🟡 у роботі |

## Що сюди НЕ вкладається

- Зміни самого алгоритму DiD, NFC compatibility, формули COEF_1 та подібні
  методологічні правки → ці зміни в `core/` + `pipeline/` з оновленням
  `docs/ALGORITHMS.md` і обов'язковим протоколом у `docs/LOGS.md`.
- Стабільні метрики, які мають з'являтися у фінальних файлах для всіх
  замовників → це частина продакшн-пайплайну (фаза C).
- Тестові артефакти, smoke-test скрипти → на час експерименту вони можуть
  бути в корені проекту з префіксом `_test_` і видаляються після.
